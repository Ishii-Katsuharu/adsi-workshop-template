# 勤怠管理システム — 要求仕様書

## 概要

中小規模（約10名）の組織向け勤怠管理Webアプリケーション。
従業員の出退勤・休憩の打刻、勤務時間の自動計算、有給休暇管理、月次レポート出力を提供する。

## ユーザーと権限

| ロール | 説明 | 主な操作 |
|--------|------|----------|
| 一般社員 | 自分の勤怠を管理する | 打刻、有給申請、自分の勤怠確認 |
| 上長 | 部下の勤怠を承認する | 勤怠承認、部下の勤怠閲覧 |
| 管理者 | システム全体を管理する | 社員登録・編集、全社員の勤怠閲覧、レポート出力、マスタ管理 |

## ユーザーストーリー

### US-1: 出勤・退勤の打刻

**一般社員として**、Web画面のボタンを押して出勤・退勤を記録したい。**それによって**、正確な勤務時間が記録される。

- ボタン押下で現在時刻を記録する
- 手動入力による時刻修正も可能（理由の記入必須）
- 事後登録・修正は当月内のみ可能
- 当日の打刻状態（未出勤 / 勤務中 / 退勤済）が画面に表示される

### US-2: 休憩時間の管理

**一般社員として**、休憩時間を管理したい。**それによって**、実労働時間が正確に計算される。

- デフォルトの休憩は 12:00〜13:00（出勤打刻時に自動設定）
- 休憩時間を変更したい場合は勤怠一覧画面から編集する（修正理由必須）

### US-3: 勤務時間の自動計算

**一般社員として**、日次・月次の勤務時間を自動計算してほしい。**それによって**、手作業による集計ミスを防げる。

- 実労働時間 = 退勤時刻 − 出勤時刻 − 休憩時間
- 所定労働時間: 9:15〜17:30（7時間15分）、休憩: 12:00〜13:00
- 残業時間 = 実労働時間 − 所定労働時間（7時間15分）。所定時間を超えた分すべてが残業

### US-4: 有給休暇の申請・管理

**一般社員として**、有給休暇を申請したい。**それによって**、休暇取得の記録と残日数を管理できる。

- 日付・種別（全日 / 午前半休 / 午後半休）・理由を指定して申請
- 上長が承認 / 却下する
- 残有給日数が表示される
- 付与: 年間20日、毎年4月1日に一括付与
- 繰越なし（年度末で未使用分は消滅）

### US-5: 勤怠承認フロー

**上長として**、部下の日次勤怠を承認したい。**それによって**、不正な勤怠記録を防止できる。

- 部下の勤怠一覧を確認し、承認 / 差し戻しを行う
- 上長は部署単位で自動決定（部署長 = 上長）
- 手動修正された打刻は修正理由とともに表示される
- 未承認の勤怠がある場合、上長に通知（画面上の件数表示のみ、メール通知なし）

### US-6: 月次勤怠レポート

**管理者として**、月次の勤怠データをCSV出力したい。**それによって**、給与計算システムへのデータ連携ができる。

- 対象月・対象社員（全員 / 部署 / 個人）を指定して出力
- 出力項目: 社員ID、氏名、出勤日数、勤務時間合計、残業時間合計、有給取得日数

### US-7: 社員管理

**管理者として**、社員情報を登録・編集したい。**それによって**、勤怠管理の対象社員を管理できる。

- 管理項目: 社員ID（自動採番）、氏名、メールアドレス、部署、役職
- 社員の追加・編集・無効化（論理削除）を行える
- 初期パスワードの設定とパスワードリセット

## 勤務ルール

| 項目 | 値 |
|------|-----|
| 所定労働時間 | 9:15〜17:30（実働 7時間15分） |
| 所定休憩 | 12:00〜13:00（1時間） |
| 残業定義 | 所定時間を超えた分すべて |
| 休日 | 土曜・日曜・祝日（固定） |

## 認証

- ログイン方式: 社員ID + パスワード
- パスワードはハッシュ化して保存（BCrypt）
- セッション管理による認証状態の維持

## 非機能要件

| 項目 | 要件 |
|------|------|
| 想定ユーザー数 | 約10名 |
| 対応デバイス | PC のみ（モバイル対応不要） |
| データ保持期間 | 3年（労働基準法に準拠） |
| 可用性 | 業務時間帯（平日 8:00〜20:00）に稼働 |
| 技術スタック | Backend: Java / Spring Boot、Frontend: TypeScript / Next.js |
