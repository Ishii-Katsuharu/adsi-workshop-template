# 画面設計

## 画面遷移図

```
ログイン ──► ダッシュボード ──┬──► 勤怠一覧
                             │
                             ├──► 有給申請
                             │
                             ├──► 承認一覧（上長・管理者）
                             │
                             ├──► 社員管理（管理者）
                             │
                             └──► レポート出力（管理者）
```

## 各画面の構成

### 1. ログイン画面

| 要素 | 説明 |
|------|------|
| 社員ID入力 | 数値入力 |
| パスワード入力 | マスク表示 |
| ログインボタン | 認証実行 |
| エラーメッセージ | 「社員IDまたはパスワードが正しくありません」 |

### 2. ダッシュボード

| 要素 | 説明 |
|------|------|
| 打刻状態表示 | 「未出勤」「勤務中」「退勤済」 |
| 出勤ボタン | 状態が未出勤のとき表示 |
| 退勤ボタン | 状態が勤務中のとき表示 |
| 今日の勤務情報 | 出勤時刻、休憩時間、現在の勤務時間 |
| 当月サマリー | 出勤日数、総勤務時間、残業時間、有給残日数 |
| 未承認バッジ | 上長のみ。承認待ち件数を表示 |

### 3. 勤怠一覧画面

| 要素 | 説明 |
|------|------|
| 年月セレクタ | 表示月の切替 |
| 勤怠テーブル | 日付、出勤、退勤、休憩開始、休憩終了、勤務時間、残業、承認状態 |
| 修正ボタン | 各行。当月内のみ活性化 |
| 修正ダイアログ | 出勤・退勤時刻 + 休憩開始・終了時刻入力 + 修正理由（必須） |
| 月合計行 | 勤務時間合計、残業合計 |

### 4. 有給申請画面

| 要素 | 説明 |
|------|------|
| 残日数表示 | 現在の有給残日数 |
| 日付選択 | カレンダー |
| 種別選択 | 全日 / 午前半休 / 午後半休 |
| 理由入力 | テキストエリア（必須） |
| 申請ボタン | バリデーション後に送信 |
| 申請履歴テーブル | 日付、種別、状態（PENDING/APPROVED/REJECTED） |

### 5. 承認一覧画面（上長・管理者）

| 要素 | 説明 |
|------|------|
| タブ切替 | 「勤怠承認」/「有給承認」 |
| 承認待ちテーブル | 社員名、日付、詳細、修正フラグ |
| 承認ボタン | 一括 or 個別 |
| 差し戻しボタン | 個別 |
| 修正理由表示 | 手動修正された行はハイライト + 理由ツールチップ |

### 6. 社員管理画面（管理者）

| 要素 | 説明 |
|------|------|
| 社員テーブル | ID、氏名、メール、部署、役職、ロール、状態 |
| 新規登録ボタン | 登録ダイアログを開く |
| 編集ボタン | 各行 |
| 無効化ボタン | 各行。確認ダイアログ付き |
| パスワードリセット | 各行 |
| 登録/編集フォーム | 氏名、メール、部署（セレクト）、役職、ロール |

### 7. レポート出力画面（管理者）

| 要素 | 説明 |
|------|------|
| 対象年月 | 年月セレクタ |
| 対象範囲 | 全員 / 部署選択 / 個人選択 |
| CSVダウンロードボタン | クリックでダウンロード開始 |
| プレビューテーブル | 出力内容のプレビュー表示 |

## 共通コンポーネント

| コンポーネント | 用途 |
|--------------|------|
| Header | ロゴ、ナビゲーション、ユーザー名、ログアウト |
| Sidebar | 画面遷移メニュー（ロールに応じて項目を出し分け） |
| StatusBadge | 承認状態の色付きバッジ |
| ConfirmDialog | 破壊的操作の確認 |
| ErrorToast | API エラー通知 |
| LoadingSpinner | 通信中表示 |
